# 更新日志

本项目遵循 [语义化版本](https://semver.org/lang/zh-CN/)。日志格式参考 [Keep a Changelog](https://keepachangelog.com/zh-CN/)。

## [未发布]

### 新增

- **批量导出图片本地化**（ROADMAP 中期项）：批量页新增「图片本地化打包」开关
  （默认跟随全局图片策略）——每个页面在自己的上下文内抓图（自动携带登录态），
  base64 经消息通道回传统一写 ZIP，目录结构与页面树导出同款
  （`assets/<md 同名目录>/`）；单页图片总量超 40MB 自动降级远程链接并在行内明示；
  各页抓图进度实时回显到对应行
- **页面树导出可选包含评论**（ROADMAP 中期项）：评论跟随 popup「导出评论」开关，
  开启时每页评论（含深层回复）随正文导出，呈现方式与单页导出一致；
  关闭开关即回到纯正文的速度；单页评论拉取失败只写入导出报告
- **URL 括号转义**：图片/链接 URL 里的字面 `()` 在提取阶段统一转为 `%28/%29`
  （百分号编码语义等价）——md 内联链接不再被 `)` 提前闭合，
  图片本地化的收集也不再截断此类 URL

### 修复（1.1.0 后全量代码审读）

- **「重新分析」真正重新提取**：INK_ANALYZE 支持 forceRefresh 绕过提取缓存——
  此前缓存键（URL+选项）不变时按钮永远命中旧缓存，页面内容变化后无法刷新
- **popup 重开恢复导出互斥**：消费 handleAnalyze 返回的 exporting 字段，
  导出进行中重开 popup 不再能触发并发二次导出
- **跨标签页消息串台**：INK_PROGRESS / INK_BUSY 按 sender.tab.id 过滤，
  批量导出等场景下其他标签页的进度/忙态不再污染当前 popup
- **代码块内容保真**：零宽字符删除、nbsp→空格、连续空行收敛全部改为围栏感知，
  不再篡改 fenced code 内部内容
- **脚注编号避让**：正文自带 `[^n]` 脚注时，织入的评论脚注从最大编号续编
- **锚点转义感知匹配**：被 Turndown 转义（`\*` `\[` 等）或跨空白/换行的划线原文
  此前必然失配降级附录，现走模糊匹配并保留转义呈现，锚定命中率显著提升；
  锚点搜索改为线性围栏扫描，未闭合代码块不再被误判为正文
- **YAML 反斜杠转义**：标题以 `\` 结尾不再损坏 front matter
- **Confluence 深层回复**：评论改走 descendant/comment 拿全所有层级
  （此前二层以上回复被静默丢弃），端点不可用自动回退旧通道
- **自定义规则选择器**：removeSel 按顶层逗号拆分，`:is(a,b)`、`[attr="x,y"]` 不再被拆坏
- **接口超时兜底**：Confluence REST 与飞书三处数据接口统一接入 30s 超时
  （fetchJsonWithTimeout 唯一实现，超时窗口覆盖到响应体读完——
  fetch 在响应头到达即 resolve，body 阶段挂起的流同样要能超时）——
  此前任一请求挂起，导出就永远停在进度态
- **预览暂存不再常驻**：预览页读取后转存本标签页 sessionStorage（刷新不丢）
  并删除 storage.local 副本
- 「导出完成」不再抢在具体结果前误报；字数统计覆盖 CJK 扩展区/假名/谚文；
  清理 background 死代码。e2e 断言 119 → 126 项

## [1.1.0] - 2026-07-07

### 新增

- **飞书文档接口化采集**：适配器从「滚动采集（实验性）」升级为「client_vars 数据接口精配」。
  不再滚动页面，直接走飞书页面自身的同源数据接口拿全量 block 树（游标自动分页拉全，
  浏览器自带登录态，插件不接触任何凭证），精确重建：标题层级、富文本（粗体/斜体/删除线/
  下划线/行内代码/链接/行内公式）、嵌套列表、有序列表序号、任务列表、引用、多行代码块
  （含语言标注）、表格（含合并单元格，接 markdown.js 现成三档策略）、mermaid 图表块、分割线、
  图片（走同源下载通道，配合本地打包）；电子表格/多维表格/画板等暂不支持的块插入占位说明并告警。
  接口成功时徽章按次升级为「精配」；接口形态变化或失败时自动回退滚动采集（保持「实验性」徽章）
  并在告警中明示原因——绝不让用户空手而归
- **飞书评论导出**：评论 id 从各内容块的 comments 字段按文档顺序收集，经 comment/batch
  接口批量换取正文——划线评论以 quote（被划线原文）锚定为「==高亮==[^脚注]」，
  全文评论进文末「💬 评论」区，回复嵌套呈现，已解决状态标注在脚注元信息里；
  @提及转为可读的 @名字、HTML 实体解码（DOMParser 惰性文档处理不可信输入）；
  拉取失败降级为仅导出正文并告警，与 Confluence 评论行为完全对齐

### 内部重构

- 设置存储收敛为 `core/settings.js` 单一实现（InkSettings：DEFAULTS / read / write / update / reset），
  替换 pipeline / popup / options / background 四处重复代码与三份默认值表——行为无变化
- 适配器提取样板下沉：`InkIR.buildContainer / normalizeContainer / pickTitle / pickText`，
  七个精配适配器与选区导出统一接入，修复各适配器清洗顺序不一致的隐患。
  注意：自定义站点规则的 `removeSel` 现在在懒加载修复**之后**执行——依赖懒加载占位状态的
  选择器（如 `img[src^="data:image/gif"]`）请改为匹配真实 src 或 `data-src` 属性
- 界面页共享层：`ui/tokens.css`（设计 token 唯一定义处，四页暗色一致性补齐）+
  `ui/shared.js`（escapeHtml 补单引号转义 / downloadBlob 统一锚点入 DOM 版本）
- 页面树导出并行化：同层子页面 3 路并发抓取，渲染与图片本地化进同一并发池重叠执行；
  对源站总并发压在 6 以内（浏览器同主机连接上限），避免触发企业站点限流
- stats() 字数统计改单趟码位计数，百万字文档不再产生海量临时分配

### 修复（合并前代码审查）

- 页面树导出触达 300 页上限后未真正停止：队列被本层子页面重新填回，继续发多余的
  REST 请求并重复写「超出上限」报告——现改为硬停（触顶层只导出、不再下钻）
- 页面树导出中单页转换/本地化异常会中断整个导出且遗留后台孤儿请求——
  现降级为该页保留远程链接并计入导出报告
- `InkSettings.update` 串行化：popup 里快速连点两个开关时，后一次读-改-写可能
  用旧快照覆盖掉前一次刚保存的偏好
- `InkSettings.DEFAULTS` 冻结（含 customRules 数组），杜绝共享默认值被调用方原地修改污染
- stats() 元素分类改用 `localName`，XHTML 页面上图片/表格数不再误计为代码块

### 修复（真机测试反馈）

- 批量导出页：长 URL 溢出容器——`text-overflow: ellipsis` 需要块级元素才生效，
  标题与链接改为 `display: block`，超长部分显示省略号
- popup 切换「导出评论」开关不再整卡闪烁：结果已在屏时改为静默重分析，
  数据回来后原地更新统计数字
- 脚注内的评论回复改为逐行缩进展示（↳ 每条一行，多行脚注续行语法，
  Obsidian/Typora/GFM 通用）——此前整条回复链用「↳」串在同一行，长讨论完全没有层级可读性

## [1.0.0] - 2026-07-07

摘墨 Inkmark 首个正式版。由原 wiki2md（仅支持 Confluence 单页导出，归档于 `legacy-v1` 分支）
完全重写而来；开发期内部迭代（2.0.0–2.4.6）的完整过程见 git 提交历史与 `docs/DESIGN.md`。

### 核心能力

- **适配器架构**：站点适配器 → 中间表示（IR）→ 唯一的 Markdown 转换器 → 输出层；
  通用网页由 Readability 兜底，任何站点都能导出
- **9 个精配适配器**：Confluence（含评论）、飞书文档（实验性）、微信公众号、知乎、
  掘金、CSDN、语雀、Stack Overflow（问题 + 全部回答结构化），另有用户自定义规则引擎
- **评论导出**：划线评论默认「==高亮==[^脚注]」锚定原文位置，页面评论与嵌套回复进
  文末「💬 评论」区；无锚点自动降级附录，三层递降不丢数据（Confluence 全支持，自动分页拉全）
- **表格三档策略**：简单表转 GFM（单元格 `|` 转义）；合并单元格网格展开后转 GFM
  （源自 v1 插件的实战方案）；嵌套表格默认保留净化 HTML，可选强制扁平化
- **公式还原**：KaTeX / MathJax v2 / 知乎公式图 → `$LaTeX$`

### 导出方式

- 单页：下载 .md / 复制剪贴板 / 图片本地化打包 ZIP / 预览页（对照·效果·源码三视图，源码可编辑实时渲染，双栏同步滚动）
- **Confluence 页面树**：当前页 + 全部子孙页面一键导出（REST 驱动、不开标签页），
  ZIP 目录镜像层级，每页图片本地化进 `assets/<页面名>/`，失败清单写入导出报告
- **多标签页批量导出**：勾选当前窗口任意页面打包 ZIP（按需授权、自动唤醒休眠标签页）
- 右键导出整页/选中节选；`Alt+Shift+M` 快捷键；导出历史（最近 30 次可找回）

### 个性化设置

- Markdown 风格：列表符号 / 强调符号 / 代码围栏 / 链接样式 / 复杂表格策略
- Front Matter 开关与自定义 tags、文件名模板（`{title}` `{domain}` `{date}`）
- 自定义站点规则（URL 包含 + CSS 选择器，无需写代码）、配置导入/导出/恢复默认

### 稳定性与安全

- 被摘录页面按不可信输入处理：危险协议链接摘除、保留 HTML 净化、预览页 DOM 级消毒
- 鉴权图片站点（Confluence/飞书/语雀）自动切换「本地打包」；HTML 块内 `<img>` 同样本地化
- 文件名 `& # %` 换全角，规避各家 Markdown 编辑器的链接目标解析差异
- SPA 缓存按 URL 失效、并发提取去重、重复注入幂等、导出互斥忙态
- 85 项端到端断言（真实 Chromium 全管线）+ 扩展加载冒烟测试
