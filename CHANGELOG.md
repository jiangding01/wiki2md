### 新增

- **X (Twitter) 适配器**（实验性，用户需求驱动）：推文详情页精配——
  主推文与同作者连续 thread 续推并入正文，回复走评论通道进文末

  滚动采集会让虚拟化时间线闪白），导出且开启评论时才自动滚动收集，
  轻量分析产物带 _lite 标记、导出时强制完整重提

- **导出历史单条删除**：每条记录新增「删除」按钮（此前只能整体清空）；
  「清空历史」补确认弹窗
- **导出历史去重**（用户反馈驱动）：同一文档（URL+标题）重复导出只保留
  最新一条，反复测试不再挤占 30 条额度；节选的价值在选区内容本身，
  同一页面的不同选区是不同产出，不参与去重

### 修复

- **一次导出触发多份下载**（用户实测反馈：飞书页面双 ZIP）：1.3.0 的
  allFrames 注入让每个 frame 都装上了消息监听器，而不带 frameId 的
  `tabs.sendMessage` 是广播——含子 frame 的页面（飞书等）一次点击会让
  顶层与子 frame 各自导出。修复分两层：所有导出/分析消息显式定向
  （顶层 frameId 0；popup 选优与右键选区定向到目标 frame 并带
  frameTargeted 标志）；页面侧新增广播守卫，子 frame 只处理显式定向
  给自己的消息——将来新增调用点忘带 frameId 也不会再重复下载
- **微信公众号新版编辑器适配**（用户实测 DOM 驱动）：
  - 图片改用 `data-src` 原图——页面 JS 会把 `src` 填成 `tp=webp&wx_lazy=1`
    的懒加载低清版，此前导出的正是这个必裂版本；同时标记 `authImages`
    （图床 referer 防盗链，远程链接在本地 md 必裂），popup 会建议本地打包
  - 样式粗体语义化：新版编辑器不再输出 `<strong>`，加粗是 `span[textstyle]`
    的 inline `font-weight`（生效值由自内向外第一个声明决定）——此前小节
    标题等加粗全部丢失；已有 strong 祖先的不重复包裹
  - 转换层新增「紧邻同类强调合并」（所有站点生效）：编辑器把一句话按样式
    段拆成多个 `<strong>` 时，Turndown 会输出解析错乱的 `**a****b**`，
    现在 DOM 层缝合为整句加粗
  - 新版正文 DOM 存入 `test/fixtures/wechat-article.html`，e2e 新增 8 项断言

## [1.3.0] - 2026-07-11

### 新增

- **iframe 正文提取**（ROADMAP 远期项）：正文位于 iframe 的页面（部分邮箱/在线
  预览器）现可自动定位并导出——注入改为 `allFrames`（跨源子 frame 无权限时静默
  降级为顶层行为），popup 向各 frame 定向分析后用新纯函数模块
  `src/core/frame-select.js`（`InkFrameSelect`）选优：顶层字数充分时直接采用，
  仅当顶层贫瘠且子 frame 显著更优（精配适配器命中或字数 ≥ 3 倍）才切换,不被
  广告/评论 iframe 误导；单帧页面走快路径，行为与旧版一致、零额外开销。
  顺带修复：右键「导出选中内容」现按 `info.frameId` 定向，子 frame 内的选区
  不再报「没有选中内容」。e2e 新增用例 19/20（11 项断言）
- **页面树导出分卷打包 + 断点续传**（ROADMAP 中期项「空间整库导出」前置）：
  遍历改为流式，每满一卷（300 页）立即打包为独立 ZIP 下载并释放内存，总量
  安全阀提至 3000 页；每卷完成时把队列游标/卷清单/失败清单持久化到
  `chrome.storage.local`，中断后 popup 检测到同根页面的未完成导出会询问
  「从第 N 卷继续 / 重新开始」，续传跳过已下载卷、重拉未完成页。单卷以内的
  小树行为与旧版完全一致（文件名不带卷号）。e2e 新增用例 21/22（20 项断言）

### 变更

- **UI 圆角体系统一**：设计 token 新增四档圆角刻度（lg 14 / md 10 / sm 8 / xs 4），
  popup/options/preview/batch 四页共 25 处硬编码圆角（3~12px 九种杂散值）全部收敛
  到刻度上——卡片与动作按钮统一 14px，分段控件按「外圆角 − 内边距」同心递减，
  整体观感更柔和统一（弹窗最外圈窗口圆角由浏览器控制，无法更改）；
  popup 面板加宽 340 → 360px，长标题与统计行更从容
- **可访问性基线**（tokens.css 四页共享）：所有可聚焦元素补 `:focus-visible`
  朱砂描边（文本输入类沿用边框高亮，不双重描边；自绘开关焦点态转移到滑轨）；
  补 `prefers-reduced-motion` 降级，动画/过渡瞬时完成
- **页面树续传询问改为面板内联**：原生 `window.confirm` 换成 popup 内的
  内联询问条（琥珀色卡片 + 「从第 N 卷继续 / 重新开始」双按钮，Esc 取消），
  不再跳出面板视觉语境
- **预览页双栏同步滚动**（ROADMAP 远期项）：由「滚动条高度比例」换算改为「源码
  行号 ↔ 渲染块」锚点级对齐——新增纯函数模块 `src/preview/preview-sync.js`
  （`InkPreviewSync`），基于 `marked.lexer()` 的 token 流 + 各 token `.raw` 的换行数
  精确还原每个顶层块在源码里的起止行号（含 front matter 行数偏移），滚动时按行号
  定位对应渲染块的 `offsetTop`、块内线性插值避免跳变；映射不可信时（空文档、顶层
  块数与实际渲染出的 DOM 节点数对不上）整体回退旧版比例同步，不会退化为不同步。
  e2e 新增用例 18（14 项断言）覆盖典型文档的行号定位、往返互逆、边界钳制与四种
  回退分支

### 内部

## [1.2.0] - 2026-07-11

### 新增

- **批量导出图片本地化**（ROADMAP 中期项）：批量页新增「图片本地化打包」开关
  （默认跟随全局图片策略）——每个页面在自己的上下文内抓图（自动携带登录态），
  base64 经消息通道回传统一写 ZIP，目录结构与页面树导出同款
  （`assets/<md 同名目录>/`）；单页图片总量超 40MB 自动降级远程链接并在行内明示；
  各页抓图进度实时回显到对应行
- **页面树导出可选包含评论**（ROADMAP 中期项）：评论跟随 popup「导出评论」开关，
  开启时每页评论（含深层回复）随正文导出，呈现方式与单页导出一致；
  关闭开关即回到纯正文的速度；单页评论拉取失败只写入导出报告
- **URL 括号转义**：图片/链接 URL 里的字面 `()` 在提取阶段统一转为 `%28/%29`
  （百分号编码语义等价）——md 内联链接不再被 `)` 提前闭合，
  图片本地化的收集也不再截断此类 URL。媒体占位（iframe/video/audio，
  不经 absolutizeUrls 的产出路径）同样转义；取舍明示：极少数按字面字节
  签名/路由的服务端（预签名 URL 等）不认编码形态，图片抓取失败时会
  按字面括号自动重试一次兜底

### 修复（导出增强分支的多智能体代码评审）

- 批量导出打包/下载阶段纳入异常保护：超大 ZIP 生成失败不再让按钮永久禁用
- 图片消息预算修正为 32MB 原始字节并常量化（原 40MB 经 base64 膨胀后
  逼近 64MB 消息上限，降级保护可能失效）
- Confluence 评论分页触达上限（1000 条）不再静默截断：单页导出告警、
  树导出写入导出报告；descendant 结果按 ancestors 过滤其他页面的评论
  （官方语义不穿越子页面，防御兜底跨版本差异）
- 树导出每页正文与评论并行拉取（开评论的大空间抓取墙钟原本接近翻倍）
- 批量页已完成行不再被迟到的进度广播覆盖；base64 转码并行化；
  文件名模板在导出时重读

### 修复（1.1.0 后全量代码审读）

- **「重新分析」真正重新提取**：INK_ANALYZE 支持 forceRefresh 绕过提取缓存——
  此前缓存键（URL+选项）不变时按钮永远命中旧缓存，页面内容变化后无法刷新
- **popup 重开恢复导出互斥**：消费 handleAnalyze 返回的 exporting 字段，
  导出进行中重开 popup 不再能触发并发二次导出
- **跨标签页消息串台**：INK_PROGRESS / INK_BUSY 按 sender.tab.id 过滤，
  批量导出等场景下其他标签页的进度/忙态不再污染当前 popup
- **代码块内容保真**：零宽字符删除、nbsp→空格、连续空行收敛全部改为围栏感知，
  不再篡改 fenced code 内部内容
- **脚注编号避让**：正文自带 `[^n]` 脚注时，织入的评论脚注从最大编号续编
- **锚点转义感知匹配**：被 Turndown 转义（`\*` `\[` 等）或跨空白/换行的划线原文
  此前必然失配降级附录，现走模糊匹配并保留转义呈现，锚定命中率显著提升；
  锚点搜索改为线性围栏扫描，未闭合代码块不再被误判为正文
- **YAML 反斜杠转义**：标题以 `\` 结尾不再损坏 front matter
- **Confluence 深层回复**：评论改走 descendant/comment 拿全所有层级
  （此前二层以上回复被静默丢弃），端点不可用自动回退旧通道
- **自定义规则选择器**：removeSel 按顶层逗号拆分，`:is(a,b)`、`[attr="x,y"]` 不再被拆坏
- **接口超时兜底**：Confluence REST 与飞书三处数据接口统一接入 30s 超时
  （fetchJsonWithTimeout 唯一实现，超时窗口覆盖到响应体读完——
  fetch 在响应头到达即 resolve，body 阶段挂起的流同样要能超时）——
  此前任一请求挂起，导出就永远停在进度态
- **预览暂存不再常驻**：预览页读取后转存本标签页 sessionStorage（刷新不丢）
  并删除 storage.local 副本
- 「导出完成」不再抢在具体结果前误报；字数统计覆盖 CJK 扩展区/假名/谚文；
  清理 background 死代码。e2e 断言 119 → 126 项

## [1.0.0] - 2026-07-07

摘墨 Inkmark 首个正式版。由原 wiki2md（仅支持 Confluence 单页导出，归档于 `legacy-v1` 分支）
完全重写而来；开发期内部迭代（2.0.0–2.4.6）的完整过程见 git 提交历史与 `docs/DESIGN.md`。

### 核心能力

- **适配器架构**：站点适配器 → 中间表示（IR）→ 唯一的 Markdown 转换器 → 输出层；
  通用网页由 Readability 兜底，任何站点都能导出
- **9 个精配适配器**：Confluence（含评论）、飞书文档（实验性）、微信公众号、知乎、
  掘金、CSDN、语雀、Stack Overflow（问题 + 全部回答结构化），另有用户自定义规则引擎
- **评论导出**：划线评论默认「==高亮==[^脚注]」锚定原文位置，页面评论与嵌套回复进
  文末「💬 评论」区；无锚点自动降级附录，三层递降不丢数据（Confluence 全支持，自动分页拉全）
- **表格三档策略**：简单表转 GFM（单元格 `|` 转义）；合并单元格网格展开后转 GFM
  （源自 v1 插件的实战方案）；嵌套表格默认保留净化 HTML，可选强制扁平化
- **公式还原**：KaTeX / MathJax v2 / 知乎公式图 → `$LaTeX$`

### 导出方式

- 单页：下载 .md / 复制剪贴板 / 图片本地化打包 ZIP / 预览页（对照·效果·源码三视图，源码可编辑实时渲染，双栏同步滚动）
- **Confluence 页面树**：当前页 + 全部子孙页面一键导出（REST 驱动、不开标签页），
  ZIP 目录镜像层级，每页图片本地化进 `assets/<页面名>/`，失败清单写入导出报告
- **多标签页批量导出**：勾选当前窗口任意页面打包 ZIP（按需授权、自动唤醒休眠标签页）
- 右键导出整页/选中节选；`Alt+Shift+M` 快捷键；导出历史（最近 30 次可找回）

### 个性化设置

- Markdown 风格：列表符号 / 强调符号 / 代码围栏 / 链接样式 / 复杂表格策略
- Front Matter 开关与自定义 tags、文件名模板（`{title}` `{domain}` `{date}`）
