## 1. 安装

摘墨目前通过「开发者模式」安装（暂未上架 Chrome 商店）。整个过程约一分钟，不需要任何技术背景。

### 1.1 拿到插件文件

两种方式任选其一：

- **方式 A（推荐给普通用户）**：找分享者要打包好的 `inkmark-vX.X.X.zip`，解压到一个**固定的文件夹**里（例如 `文档/摘墨插件/`）。
  > ⚠️ 注意：安装后这个文件夹不能删除或移动，浏览器每次启动都会从这里加载插件。
- **方式 B（会用 git 的用户）**：`git clone` 仓库到本地，之后更新只需 `git pull`。

### 1.2 加载到 Chrome

1. 在地址栏输入 `chrome://extensions` 并回车
2. 打开右上角的「**开发者模式**」开关
3. 点击左上角「**加载已解压的扩展程序**」
4. 选择刚才解压（或 clone）的文件夹
5. 看到朱砂印章「摘」图标出现在列表里即安装成功

> 📸 **截图占位 01**：chrome://extensions 页面，标出「开发者模式」开关和「加载已解压的扩展程序」按钮的位置

建议顺手做一件事：点击浏览器工具栏的拼图图标 🧩，把「摘墨」**固定**到工具栏上，之后一键可达。

> 📸 **截图占位 02**：工具栏拼图菜单中固定摘墨的操作

### 1.3 更新插件

- zip 方式：用新版本 zip 的内容**覆盖**原文件夹，然后到 `chrome://extensions` 点摘墨卡片上的刷新按钮 ⟳
- git 方式：`git pull` 后同样点刷新按钮 ⟳

## 2. 快速上手：三步导出你的第一篇文档

1. **打开**任意一篇你想保存的网页（文章、文档、帖子都行）
2. **点击**工具栏上的朱砂印章图标——摘墨会自动分析页面，一两秒内显示出标题、字数、图片数等统计
3. **点击**红色的「下载 Markdown」大按钮——搞定，.md 文件已在你的下载目录里

> 📸 **截图占位 03**：在一篇文章页上打开 popup，面板显示分析结果，光标悬停在「下载 Markdown」按钮上

下载的文件可以直接拖进 Obsidian、Typora、VS Code 等任何 Markdown 工具阅读和编辑。

## 3. 主面板详解

点击工具栏图标打开的面板是摘墨的主界面，从上到下分为三块：**状态区 → 选项区 → 动作区**。

> 📸 **截图占位 04**：popup 全貌，用标注框分别圈出状态区、选项区、动作区

### 3.1 状态区

| 元素 | 说明 |
|---|---|
| **适配器徽章** | 显示当前页面由哪个「适配器」处理。**精配**（如 Confluence、飞书文档）= 摘墨对该站点做了专门优化，导出质量最好；**实验性** = 已适配但站点结构多变，可能偶有瑕疵；**通用模式** = 没有专门适配，走智能正文识别兜底，绝大多数文章页效果依然很好 |
| **标题** | 导出文件的标题，**可以直接点击修改**——改动会同时作用于文件名、Front Matter 和正文的 # 一级标题 |
| **统计行** | 字数 / 图片 / 表格 / 代码块数量；页面有评论时还会显示评论数。用它快速判断「抓到的内容对不对」 |
| **提示条** | 淡黄色区域，显示需要你知道的信息：比如「该站点图片需登录，已自动切换为本地打包」「本页正文几乎为空——可能是目录页」等 |
| **重新分析 ⟳** | 右上角的小刷新图标。页面内容变化后（比如刚展开了折叠区块）点它重新提取 |

### 3.2 选项区

| 选项 | 说明 |
|---|---|
| **YAML Front Matter** | 是否在文件开头写入元数据块（标题/来源链接/作者/摘录时间/标签）。用 Obsidian 的强烈建议开启 |
| **导出评论** | 是否把页面上的评论一起导出（详见[第 5 章](#5-评论导出)）。关闭后导出更快 |
| **图片** | 「远程链接」或「本地打包」二选一（详见[第 6 章](#6-图片远程链接还是本地打包)） |

这些开关的选择会被记住，下次打开保持不变。

### 3.3 动作区

| 按钮 | 行为 |
|---|---|
| **下载 Markdown / 下载 ZIP** | 主按钮。图片选「远程链接」时下载单个 .md；选「本地打包」时下载 ZIP（内含 .md + assets 图片文件夹） |
| **复制** | 把 Markdown 全文复制到剪贴板，方便直接粘贴到笔记软件 |
| **预览** | 新标签页打开预览（详见[第 7 章](#7-预览页)），导出前先看效果 |
| **导出页面树** | 仅在 Confluence 页面出现（详见[第 8 章](#8-confluence-页面树导出)） |

导出进行中所有按钮会临时禁用并显示进度（如「抓取图片 12/41…」），防止重复触发。

## 4. 支持的网站

### 4.1 精配站点

以下站点做了专门优化，导出质量最好：

| 站点 | 特色能力 |
|---|---|
| **Confluence** | 代码宏/信息面板规范转换；评论完整导出（划线评论转脚注）；整棵页面树批量导出 |
| **飞书文档** | 走数据接口精确重建全文（标题层级/表格含合并单元格/代码块/公式/任务列表/mermaid 图）；评论导出；接口失败自动回退滚动采集 |
| **微信公众号** | 原图导出（自动处理防盗链，建议本地打包）；新版编辑器的加粗样式正确还原 |
| **知乎** | 专栏与回答；公式图自动还原为 LaTeX 源码 |
| **掘金 / CSDN / 语雀** | 代码块装饰控件清理、折叠展开、专有标签还原 |
| **Stack Overflow** | 问题 + 全部回答结构化导出（含得分与采纳标记） |
| **X (Twitter)** | 普通推文：主推文 + 同作者连发（thread）为正文，回复进评论区；长文 Article：完整正文（标题/列表/代码块/配图说明）。需登录 X 才能看到回复 |

### 4.2 任意其他网页

没有专门适配的网站走「通用模式」：用与 Firefox 阅读模式同款的正文识别引擎提取，博客、新闻、文档类页面效果都不错。

另外两项通用能力：

- **iframe 正文提取**：正文藏在内嵌框架里的页面（网页邮箱、在线预览器）会自动定位到正确的框架
- **自定义站点规则**：内置效果不好的站点，你可以自己配一条规则精配它（见[第 13 章](#13-自定义站点规则让内网系统也能精配)）

## 5. 评论导出

摘墨对评论的处理是特色能力，分两种类型：

### 5.1 划线评论（Confluence / 飞书）

原文里被划线评论的位置，导出后正文中呈现为 **==高亮==** + 脚注序号：

```markdown
支付网关承载了 ==12 条业务线== [^1] 的交易流量。

[^1]: **陈默 · 2026-07-05**：这个数字含重试流量，去重后约 1.9 亿。
    ↳ **苏澄**：已修正
```

在 Obsidian/Typora 里，高亮直接可见、点击脚注跳转评论内容——「哪句话被谁评论了什么」完整保留。

> 📸 **截图占位 05**：Obsidian 中打开一份带划线评论的导出文件，高亮与脚注同屏可见

### 5.2 页面评论

文档底部的普通评论，统一汇入文末的「💬 评论」区，回复以嵌套引用呈现，自动分页拉全。

### 5.3 相关设置

- popup 的「导出评论」开关控制**是否导出**
- 设置页的「评论呈现」控制**怎么呈现**：`脚注+附录`（默认）/ `仅脚注` / `全部进附录`
- 设置页可单独关闭 ==高亮==（部分平台如 GitHub 不认这个语法，会显示字面等号）

## 6. 图片：远程链接还是本地打包？

| 策略 | 产物 | 适用场景 |
|---|---|---|
| **远程链接** | 单个 .md，图片仍指向原网址 | 公开网站的文章，图床可直接访问 |
| **本地打包** | ZIP：`.md + assets/ 图片文件夹` | **需要登录的站点**（Confluence/飞书/语雀）和**有防盗链的站点**（微信公众号）——远程链接在本地必然裂图，必须打包 |

摘墨检测到鉴权/防盗链站点会**自动切换到本地打包**并在提示条说明（只影响本次，可手动切回）。

本地打包时图片在页面内抓取，自动带上你的登录态，4 路并发、单张 20 秒超时；个别抓取失败的图会保留远程链接并在完成提示中告知，绝不静默丢失。

解压 ZIP 后 `.md` 与 `assets/` 保持相对引用，整个文件夹拖进 Obsidian 库即可离线阅读。

## 7. 预览页

点击 popup 的「预览」按钮，新标签页打开三视图预览：

> 📸 **截图占位 06**：预览页「对照」视图全貌，左侧渲染效果、右侧 Markdown 源码

- **对照**（默认）：左侧渲染效果 + 右侧源码，双栏**同步滚动**（滚哪边另一边都跟着走，精确到段落级）
- **效果**：只看渲染结果，接近最终在 Obsidian 里的样子
- **源码**：只看 Markdown 文本

两个实用细节：

1. **源码可以直接编辑**——改动实时重新渲染，右上角「下载 .md」和「复制 Markdown」拿到的都是你编辑后的版本。适合导出前顺手删掉不想要的段落
2. Front Matter 元数据不混在正文里，而是渲染为顶部的信息标签条（title / source / author / clipped 等）

## 8. Confluence 页面树导出

在 Confluence 页面上，popup 会多出一个「🌳 导出页面树」按钮——把**当前页 + 全部子孙页面**一键导出为 ZIP，目录结构与页面树层级完全一致。

> 📸 **截图占位 07**：Confluence 页面上的 popup，标出「导出页面树」按钮

特性一览：

- **全程走接口**，不会打开任何新标签页，速度快
- **目录镜像层级**：`根页面.md`、`根页面/子页面.md`、`根页面/子页面/孙页面.md`……
- **图片全部本地化**：每层目录一个 `assets/`，按页面名分组，解压即可离线阅读
- **评论跟随开关**：popup 的「导出评论」开着，每页评论就随正文导出
- **大空间分卷**：超过 300 页自动拆成多个 ZIP 陆续下载（总量上限 3000 页）
- **断点续传**：中途关页面/断网了？重新打开 popup 会询问「从第 N 卷继续」，不用从头再来
- **导出报告**：个别页面抓取失败不会中断整体，失败清单写进 ZIP 内的「导出报告.md」

> 📸 **截图占位 08**：解压后的页面树 ZIP 在访达/资源管理器中的目录结构

## 9. 批量导出多个标签页

开了一堆文章标签页想一次性全部存下来？点 popup 底部的「📚 批量导出标签页」。

> 📸 **截图占位 09**：批量导出页面，显示标签页勾选列表与「图片本地化打包」开关

使用流程：

1. **首次使用**需要授权「访问所有网站」权限（因为要向多个标签页注入提取脚本）——点「授予权限」即可，不授权不影响单页导出功能
2. 勾选要导出的标签页（默认全选；休眠中的标签页会标注「休眠」，导出时自动唤醒）
3. 按需勾选「**图片本地化打包**」——内网 wiki、飞书这类需要登录的页面建议勾上，图片会在各自页面内抓取后统一打包
4. 点「导出所选为 ZIP」——每行实时显示进度和结果（✓ 成功 / ✓ N 图 / ✗ 失败原因），单页失败不影响其余页面

产物是一个 ZIP：每页一个 .md，本地化的图片按 `assets/<文件名>/` 分组。

## 10. 右键节选与快捷键

**右键节选**：选中页面上的任意内容 → 右键 →「摘墨：导出选中内容为 Markdown」——只导出你选中的部分，适合摘录金句、保存半篇文章。

> 📸 **截图占位 10**：选中一段文字后的右键菜单，标出摘墨的两个菜单项

**右键整页**：不选中任何内容直接右键，菜单里有「摘墨：导出本页为 Markdown」。

**快捷键**：`Alt + Shift + M` 一键下载当前页（不用打开面板）。

这两种「无界面」导出的结果通过工具栏图标的**角标**反馈：绿色 ✓ = 成功，红色 ! = 失败。图片策略跟随你在设置页配置的默认值。

## 11. 导出历史

设置页的「导出历史」标签（popup 底部「🕘 导出历史」直达）记录最近 30 次导出：

> 📸 **截图占位 11**：导出历史列表，标出复制/下载/原文/删除四个按钮

- 每条记录可**复制**内容、重新**下载**、跳回**原文**页面、单条**删除**
- **同一文档重复导出只保留最新一条**（节选除外），反复调试不会刷屏
- 记录仅存在你自己的电脑上；单篇超过 300KB 的只保留标题等元信息
- 「清空历史」会二次确认，不怕误触

## 12. 设置页

右键工具栏图标 →「选项」，或点 popup 右上角齿轮进入。四个标签页：

### 通用

| 设置 | 说明 |
|---|---|
| Front Matter 及自定义标签 | 元数据开关；`tags` 默认 `clippings`，可改成你的归档体系（逗号分隔多个） |
| 正文包含标题 | 是否在正文开头加 `# 标题` |
| 评论呈现方式 | 脚注+附录 / 仅脚注 / 全部附录；高亮开关 |
| 图片默认策略 | 远程链接 / 本地打包（popup 里的选择会同步到这里） |
| 文件名模板 | 支持 `{title}`、`{domain}`、`{date}` 三个变量，如 `{date}-{title}` |
| 导出历史开关 | 不想留记录可以关掉 |

### Markdown 风格

列表符号（`-`/`*`/`+`）、强调符号（`*`/`_`）、代码围栏（``` 或 ~~~）、链接形式（内联/引用式）——按你的笔记软件习惯调整。

**复杂表格策略**（三档）：

| 档位 | 行为 |
|---|---|
| 智能（默认） | 合并单元格自动展开成规则表格转 Markdown，只有嵌套表格保留 HTML |
| 保守 | 任何复杂表格都保留净化后的 HTML，结构零丢失（Obsidian/Typora 都能渲染） |
| 强制纯 MD | 一律转 Markdown 表格，复杂结构有损压平（适合只认纯 md 的系统） |

### 站点规则

见[第 13 章](#13-自定义站点规则让内网系统也能精配)。

### 配置备份

- **导出配置**：把全部设置（含自定义规则）存成 JSON 文件
- **导入配置**：换电脑、或把调好的站点规则分享给同事
- **恢复默认**：一键重置（导出历史不受影响）

## 13. 自定义站点规则：让内网系统也能精配

某个网站用通用模式导出总带着导航栏、推荐位等噪音？不用写代码，配一条规则就能精配它。

> 📸 **截图占位 12**：设置页「站点规则」标签，一条填写完整的规则卡片

规则只有两个必填项：

| 字段 | 填什么 | 示例 |
|---|---|---|
| **URL 包含**（必填） | 网址里独有的一段文字 | `wiki.mycompany.com` |
| **正文选择器**（必填） | 正文区域的 CSS 选择器 | `#main-content`、`.article-body` |
| 标题选择器（可选） | 标题元素的选择器，不填用网页标题 | `h1.title` |
| 剔除选择器（可选） | 要删掉的噪音区域，逗号分隔 | `.ads, .recommend, .toolbar` |

**怎么找正文选择器**（一次学会）：

1. 在目标页面的正文上**右键 →「检查」**，开发者工具会定位到对应元素
2. 往上找几层，找到那个**包住整篇正文**的元素，看它的 `id` 或 `class`
3. 有 id 就填 `#id值`；有 class 就填 `.class值`

> 📸 **截图占位 13**：开发者工具中定位正文容器元素，标出 id 属性

保存后刷新目标页面，popup 徽章会显示你的规则名——自定义规则的优先级**高于所有内置适配器**。选择器万一失效，摘墨会自动回退到通用模式并提示你，绝不会导不出东西。

## 14. 常见问题

**Q：面板显示「0 字」，提示"本页正文几乎为空"？**
这个页面本身可能就是目录页/封面页（比如 Confluence 空间首页）。如果是 Confluence，直接用「导出页面树」打包全部子页面。

**Q：导出的 Markdown 里图片全裂？**
图片在需要登录或有防盗链的站点上，远程链接在本地打不开。把图片策略切成「**本地打包**」重新导出即可（这类站点摘墨通常会自动切换并提示）。

**Q：X (Twitter) 点下载时页面闪了一下？**
正常现象。X 的评论是滚动加载的，导出时摘墨会自动滚动页面收集回复（伴随「正在收集回复」进度提示），滚动过程中页面会重绘一次。不需要评论时关掉「导出评论」开关，导出就不会滚动。

**Q：评论没有导出来？**
依次检查：① popup 的「导出评论」开关是否开启；② 该站点是否支持评论导出（目前：Confluence、飞书、X）；③ X 需要登录后才能看到回复。

**Q：想导出本地的 HTML 文件（file:// 开头），提示无法访问？**
到 `chrome://extensions` → 摘墨 → 详情 → 开启「**允许访问文件网址**」。

**Q：在 Chrome 应用商店、chrome:// 设置页上用不了？**
浏览器不允许任何扩展访问这些系统页面，属正常限制。

**Q：飞书文档徽章显示「实验性」而不是「精配」？**
说明本次接口采集失败、自动回退了滚动采集（告警里有原因）。内容仍会导出，但复杂表格可能降级。刷新页面重试通常可恢复精配。

**Q：导出的表格在我的工具里显示不对？**
去设置页调整「复杂表格策略」：Obsidian/Typora 用户推荐默认的「智能」档；粘贴到只支持纯 Markdown 的系统就切「强制纯 MD」。

**Q：安装后图标是灰色的 / 点了没反应？**
确认加载的是**解压后的文件夹**（里面直接能看到 manifest.json），而不是 zip 本身；文件夹被移动或删除后需要重新加载。

## 15. 隐私说明

- 摘墨**没有任何服务器**，不上传、不收集任何数据——提取、转换、打包全部在你的浏览器本地完成
- 只在你主动点击（面板/右键/快捷键）时才读取当前页面，平时不监听任何网页
- 抓取需登录的图片时使用的是你浏览器自己的登录态，插件不接触、不存储任何账号凭证
- 导出历史与设置仅存于本机浏览器，可随时清空
