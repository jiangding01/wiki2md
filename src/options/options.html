摘墨 · 设置

这里的选择会成为 popup、右键菜单与快捷键导出的默认行为

导出内容

Front Matter 标签 写入 tags: 字段，逗号分隔多个

评论

评论呈现方式 脚注：划线评论锚定在原文位置；附录：统一放在文末「💬 评论」区

图片与文件

图片默认策略 本地打包：并发抓取图片存入 assets/ 并压缩为 ZIP（需要登录态的图片也能带走）
文件名模板 可用变量：{title} 标题 · {domain} 域名 · {date} 日期

语法偏好

无序列表符号 影响所有 ul 列表项的前缀
强调符号 斜体/加粗使用 * 或 _
代码块围栏 fenced code block 的围栏字符
链接样式 内联：[文字](url)；引用：[文字][1] 并在文末集中列出

高亮（==文本==）、脚注、callout 引用块等增强语法始终启用，兼容 Obsidian / Typora。

自定义站点规则

内置适配器效果不佳，或想精配某个内网系统？添加一条规则即可，无需写代码。 命中规则的页面优先使用你的选择器提取正文；选择器失配时自动回退通用模式，不会导出失败。

示例

规则名公司内部 GitLab Wiki
URL 包含gitlab.mycorp.com（命中所有含此子串的地址）
正文选择器.wiki-content（右键正文 → 检查，找到包住全文的元素）
标题选择器h1.wiki-title（可留空，默认用网页标题）
剔除选择器.sidebar, .edit-btn（逗号分隔，可留空）

最近导出

最近 30 次导出记录，仅存于本机。正文过大（>300KB）的记录只保留元信息。
